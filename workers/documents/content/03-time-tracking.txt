TIME TRACKING SYSTEM
====================

The core functionality of the Workers Portal - tracking work hours with precision and context.

Clock Widget Features
---------------------

CLOCK IN/OUT
- One-click clock in to start a session
- Automatic timestamp recording
- Clock out with optional notes
- Running timer display (HH:MM:SS)

PROJECT ASSIGNMENT
- Assign time to specific projects
- Color-coded project indicators
- "No Project" option for general work
- Can change project mid-session

BREAK MANAGEMENT
- Pause timer for breaks
- Break duration tracking
- Automatic break calculation
- Multiple breaks per session supported

SESSION NOTES
- Add context to time entries
- Searchable notes history
- Saved to database for reporting
- Optional - not required

API Endpoints
-------------
POST /api/clock-in
  - Creates new time entry
  - Records start timestamp
  - Accepts project_id (optional)

POST /api/clock-out
  - Updates current entry with end time
  - Calculates total break minutes
  - Accepts notes (optional)

POST /api/break/start
  - Records break start time
  - Links to current time entry

POST /api/break/end
  - Records break end time
  - Calculates break duration

Data Model
----------
time_entries:
  - id: Primary key
  - user_id: Foreign key to users
  - project_id: Foreign key to projects (nullable)
  - clock_in: DateTime (required)
  - clock_out: DateTime (nullable until clocked out)
  - break_minutes: Integer (calculated)
  - notes: Text (optional)
  - created_at: DateTime
  - updated_at: DateTime

breaks:
  - id: Primary key
  - time_entry_id: Foreign key
  - start_time: DateTime
  - end_time: DateTime (nullable until break ends)

Calculations
------------
Net Hours = (clock_out - clock_in) - break_minutes

Weekly Hours = SUM of all net hours for current week (Mon-Sun)

Daily Hours = SUM of all net hours for current day
